# Спецификация: Кабинет Рекрутера (Recruiter Dashboard)

## 1. Обзор модуля
Кабинет рекрутера — это основной рабочий инструмент для образовательных агентов. Здесь они ведут базу своих клиентов (студентов), ищут подходящие программы, подают заявки и отслеживают свои доходы.

## 2. Основные разделы и функции

### 2.1. Управление студентами (Student CRM)
Внутренняя CRM-система рекрутера для ведения профилей студентов.
- **Список студентов:** Карточки или таблица с базовой информацией (Имя, Гражданство, Возраст).
- **Профиль студента:**
  - Личные данные (ФИО, Email, Дата рождения).
  - **Хранилище документов:** Загрузка сканов паспорта, дипломов, языковых сертификатов. Файлы загружаются в `Supabase Storage` (бакет `student_documents`).
  - **История заявок:** Список всех программ, на которые был подан данный студент, с их текущими статусами.

### 2.2. Каталог программ (Program Catalog)
Глобальный поиск по всем активным (`status = 'Active'`) программам на платформе.
- **Поиск и Фильтрация:**
  - По названию программы или учебного заведения.
  - По стране обучения.
  - По специализации (использование компонента `SpecializationSelector`).
  - По уровню (Bachelor, Master).
  - По стоимости (Min/Max Tuition Fee).
- **Карточка программы:** Отображает название, школу, стоимость, требования (язык, опыт) и **размер комиссии** (Commission %), который получит рекрутер.
- **Действие "Apply":** При нажатии открывается модальное окно выбора студента из базы рекрутера для подачи заявки.

### 2.3. Мои заявки (My Applications)
Отслеживание статуса поданных заявок.
- **Интерфейс:** Kanban-доска с колонками:
  - `Submitted`: Заявка отправлена, школа еще не взяла в работу.
  - `Under Review`: Школа рассматривает документы.
  - `Accepted`: Студент зачислен (получен Offer).
  - `Paid`: Студент оплатил обучение, комиссия начислена.
  - `Rejected`: Отказ.
- **Взаимодействие:** Из карточки заявки можно в один клик перейти в чат со школой.

### 2.4. Сообщения (Inbox)
Связь с учебными заведениями.
- **Список чатов:** Слева отображается список чатов. Название чата — это название школы. Под названием указано имя студента и ID заявки.
- **Интерфейс чата:**
  - Отправка текстовых сообщений.
  - Прикрепление дополнительных документов (если школа запросила их в ходе `Under Review`).
  - Возможность позвать администратора (кнопка `Add Admin`), если школа не отвечает или возник спор.

### 2.5. Финансы и Баланс (Wallet / Balance)
Управление заработанными средствами.
- **Метрики:**
  - `Pending Balance`: Сумма комиссий по заявкам в статусе `Accepted` (ожидается оплата от студента).
  - `Available Balance`: Сумма комиссий по заявкам в статусе `Paid` (можно выводить).
- **История транзакций:** Список начислений с указанием ID заявки и имени студента.
- **Вывод средств (Withdrawal):**
  - Кнопка `Request Payout`.
  - Ввод суммы и банковских реквизитов (IBAN, SWIFT).
  - Создается запись в таблице `payout_requests` со статусом `Pending`. Баланс `Available` замораживается на эту сумму.

## 3. Технические особенности (RLS & Security)
- **Изоляция данных:** Рекрутер видит **только** своих студентов (`recruiter_id = auth.uid()`) и **только** свои заявки.
- **Защита Storage:** Доступ к файлам в бакете `student_documents` разрешен только владельцу-рекрутеру, школе (к которой подана заявка) и администратору.
- **UI Компоненты:** Цветовая схема базируется на цвете `blue-600`. Используются скелетоны при загрузке каталога программ.
