# Обзор платформы и Архитектура (Education Recruitment B2B Platform)

## 1. Концепция продукта
Платформа представляет собой B2B SaaS-маркетплейс, объединяющий учебные заведения (Institutions) и образовательных агентов/рекрутеров (Recruiters). 
- **Учебные заведения** публикуют свои образовательные программы и получают поток верифицированных заявок от студентов со всего мира.
- **Рекрутеры** ведут базу своих студентов, подбирают им программы, подают заявки и получают комиссионное вознаграждение за каждого успешно зачисленного и оплатившего обучение студента.
- **Администраторы (Платформа)** модерируют участников, устанавливают размер комиссий, решают спорные ситуации и управляют финансовыми потоками.

## 2. Технологический стек
- **Frontend Framework:** React 18+ (Functional Components, Hooks).
- **Build Tool:** Vite.
- **Language:** TypeScript (строгая типизация интерфейсов и моделей данных).
- **Styling:** Tailwind CSS (utility-first подход).
- **Animations:** `motion/react` (Framer Motion) для плавных переходов между вкладками и модальными окнами.
- **Icons:** `lucide-react`, `country-flag-icons` (для отображения флагов стран).
- **Backend & Database:** Supabase (PostgreSQL).
- **Authentication:** Supabase Auth (Email/Password, JWT).
- **Real-time:** Supabase Channels (для системы чатов).
- **Storage:** Supabase Storage (для хранения документов студентов, аватаров, логотипов).
- **Date/Time Management:** `moment`, `moment-timezone`, `date-fns`.

## 3. Ролевая модель (RBAC)
Система использует строгую ролевую модель на базе поля `role` в таблице `profiles`.

### 3.1. Admin (Администратор)
- Высший уровень доступа.
- Верификация новых рекрутеров (KYC).
- Модерация новых образовательных программ (установка % комиссии).
- Мониторинг всех заявок и транзакций.
- Доступ к любому чату для решения споров (Dispute Resolution).
- Обработка запросов на выплату средств (Payouts).

### 3.2. Recruiter (Рекрутер / Агент)
- Управление базой студентов (добавление, загрузка документов).
- Поиск программ по каталогу (фильтрация по странам, специальностям, ценам).
- Подача заявок от имени студентов.
- Общение со школами напрямую через встроенный чат.
- Отслеживание своего баланса и запрос на вывод заработанных комиссий.

### 3.3. Institution (Учебное заведение)
- Управление профилем школы.
- Создание и редактирование образовательных программ.
- Обработка входящих заявок (воронка: Submitted -> Under Review -> Accepted -> Paid).
- Общение с рекрутерами по конкретным заявкам (запрос недостающих документов).

## 4. Структура проекта (Frontend)
```text
/src
  /components
    /auth          # Компоненты авторизации и регистрации
    /dashboard     # Основные компоненты личных кабинетов (Inbox, ProgramsList, etc.)
    /layout        # Навигация, Sidebar, Header
    /ui            # Переиспользуемые UI элементы (MultiSelect, SpecializationSelector)
  /constants       # Статические данные (countries.ts, specializations.ts)
  /types           # TypeScript интерфейсы (index.ts)
  /lib             # Утилиты
  App.tsx          # Корневой компонент, роутинг на основе состояния авторизации
  supabase.ts      # Инициализация клиента Supabase
```

## 5. Общие UI/UX Принципы
- **Минимализм и чистота:** Использование светлых тонов (bg-gray-50, bg-white), скругленных углов (rounded-2xl, rounded-3xl) и мягких теней (shadow-sm).
- **Цветовое кодирование ролей:**
  - Admin: Indigo (`indigo-600`)
  - Institution: Emerald (`emerald-600`)
  - Recruiter: Blue (`blue-600`)
- **Отзывчивость (Responsiveness):** Интерфейс должен корректно работать на десктопах и планшетах. Таблицы имеют горизонтальный скролл.
- **Состояния загрузки (Loading States):** Использование спиннеров и скелетонов при ожидании ответа от Supabase.
- **Обработка ошибок:** Понятные уведомления (Toast/Alert) при неудачных сетевых запросах.
