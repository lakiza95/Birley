# Спецификация: Кабинет Учебного Заведения (Institution Dashboard)

## 1. Обзор модуля
Личный кабинет учебного заведения является ключевым интерфейсом для поставщиков образовательных услуг на платформе. Модуль предназначен для управления образовательными программами, обработки заявок от студентов (через рекрутеров) и коммуникации с агентами и администрацией платформы.

## 2. Ролевая модель: Institution
Пользователь с ролью `institution` имеет доступ к данным только своего заведения (`auth.uid() = institution_id`). 
**Основные права:**
- Просмотр и редактирование профиля заведения.
- Создание, редактирование и архивация образовательных программ.
- Управление статусами заявок студентов, поданных на программы заведения.
- Ведение переписки в чатах, привязанных к заявкам.

## 3. Функциональные требования

### 3.1. Панель управления (Dashboard Overview)
- **Виджеты статистики:**
    - Общее количество активных программ.
    - Количество заявок в статусе "Submitted" (требующих внимания).
- **Графики:** Визуализация динамики подачи заявок за последние 30 дней.

### 3.2. Управление программами (Programs Management)
Интерфейс для работы со списком программ заведения.
- **Список программ:** Таблица с колонками: Название, Специализация, Уровень, Длительность, Интейк, Статус (Active/Pending/Rejected), Стоимость обучения.
- **Создание/Редактирование программы:**
    - **Название:** Текстовое поле.
    - **Уровень:** Выбор из списка (Bachelor, Master, PhD, Vocational, и т.д.).
    - **Специализация:** Использование компонента `SpecializationSelector` (иерархический мультиселект с категориями и поиском).
    - **Даты:** Дата начала, дата окончания, дедлайн подачи документов.
    - **Требования (Requirements):**
        - Минимальный и максимальный возраст.
        - Чекбокс "Language Certificate Required". При активации динамически появляется поле "Min Score".
        - Чекбокс "Experience Required".
    - **География (Использование компонента `MultiSelect`):** 
        - `Countries Preferred`: Страны, из которых школа хочет видеть студентов (с поиском и флагами).
        - `Countries Not Accepted`: Страны, студенты из которых не принимаются (с поиском и флагами).
    - **Документы:**
        - `Required Documents`: Список документов, которые студент обязан предоставить для подачи заявки (Passport, Transcript, IELTS и т.д.). Используется компонент `MultiSelect` со списком типов документов.
    - **Финансы:** Стоимость обучения (Tuition Fee) и выбор валюты.
    - **Комиссия:** Поле `commission` скрыто от школы при создании. Его устанавливает только Admin.

### 3.3. Обработка заявок (Applications Processing)
- **Список заявок:** Фильтрация по статусам (Submitted, Under Review, Accepted, Rejected).
- **Детали заявки:** Просмотр профиля студента (ФИО, гражданство, возраст, сканы документов из Supabase Storage).
- **Управление статусом:** 
  - Перевод из `Submitted` в `Under Review`.
  - Перевод в `Accepted` (выдача оффера).
  - Перевод в `Paid` (подтверждение получения оплаты от студента). Это действие триггерит начисление комиссии рекрутеру.
  - Перевод в `Rejected` с обязательным указанием причины.
- **Переход в чат:** Кнопка быстрого перехода в Inbox к диалогу по данной заявке.

### 3.4. Сообщения (Inbox)
- **Список чатов:** Отображение активных диалогов. Каждый чат привязан к конкретной заявке. Название чата — имя рекрутера/агентства.
- **Интерфейс чата:**
    - Real-time обмен сообщениями (Supabase Channels).
    - Отображение контекста в шапке (Имя студента, ID заявки).
    - Системные уведомления (например, `__SYSTEM_NOTIFICATION__ Administrator has joined the chat`).
    - Возможность запросить дополнительные документы через чат.

### 3.5. Настройки профиля (Profile Settings)
- Редактирование данных заведения: Название, Тип (Private/Public), Контактный email, Страна, Город.
- Настройка часового пояса (Timezone) с отображением смещения GMT (например, `Europe/London (GMT+01:00)`).

## 4. Технические требования и интеграция

### 4.1. База данных (Supabase/PostgreSQL)
- Использование массивов `text[]` для хранения выбранных специализаций и стран (`countries_preferred`, `countries_not_accepted`).
- Настройка RLS: `auth.uid() = institution_id`. Школа не должна видеть программы и заявки других школ.

### 4.2. UI/UX
- **Цветовая схема:** Основной цвет Emerald (`emerald-600`) для элементов интерфейса школы.
- **Адаптивность:** Корректное отображение таблиц на планшетах (горизонтальный скролл).
- **Флаги:** Использование библиотеки `country-flag-icons` внутри компонента `MultiSelect`.
